### `ETLProcessor` class (Modal Class with lifecycle hooks)
- **`@modal.enter()` `startup`** — reads `DB_HOST`/`DB_PORT` from the secret and sets the staging directory path
- **`@modal.exit()` `shutdown`** — graceful teardown log
- **`extract(batch_id)`** — writes raw CSV-style data to `/mnt/staging/raw_batch_<id>.txt` on the NFS
- **`transform(batch_id)`** — reads raw file, uppercases and enriches it, writes to `/mnt/staging/transformed_batch_<id>.txt`
- **`load(batch_id)`** — reads transformed file and simulates a DB load to `DB_HOST:DB_PORT`
- **`run_batch(batch_id)`** — orchestrates extract → transform → load for one batch

### Scheduling & entrypoints
- **`scheduled_pipeline`** — `@app.function(schedule=modal.Cron("0 * * * *"))` triggers the pipeline every hour
- **`main` local entrypoint** — processes batches `[1, 2]`, then writes to the Dict:
  - `last_run_status = "success"`
  - `batches_processed = 2`
