### How rate-limiting works

1. Each POST request must include `{"api_key": "gateway-key-prod-2024", "payload": {...}}`
2. The key is validated against `GATEWAY_API_KEY` from the secret
3. Per-key request count is tracked in the Dict under key `rk:<api_key>`
4. If count ≥ `RATE_LIMIT_MAX` (100), returns HTTP 429 and increments `rate_limit_hits`
5. Otherwise, increments `total_requests_processed` and the per-key counter
