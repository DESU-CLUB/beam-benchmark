### What the pipeline does end-to-end (`@app.local_entrypoint`)

1. **Parallel batch processing** — `DocumentProcessor.process.map()` fans out 7 base64 documents to remote containers simultaneously; each container's `@modal.enter` hook initialises the parser once before handling requests
2. **Per-document stats** — each call decodes the base64 string and computes `word_count`, `sentence_count`, and `char_count` using only stdlib regex/collections
3. **Secret reading** — every worker reads `OCR_SERVICE_KEY` from `os.environ` and reports its presence
4. **Aggregation** — results are collected and summed locally
5. **Volume write** — `DocumentProcessor.write_summary` writes `summary.json` to `/ocr_output` inside the Modal Volume and calls `volume.commit()`
6. **Dict update** — all five required keys are written to `modal-doc-ocr-pipeline-output`
