### Modal primitives

### `ETLPipeline` class (`@app.cls`)
- **`@modal.enter()` → `setup()`** — lifecycle hook; reads `DB_URL` / `API_TOKEN` from the secret and mints a `run_id` for the session.
- **`extract()`** — simulates pulling 100 records from an upstream source.
- **`transform()`** — filters to even-id rows and uppercases labels (50 records pass through).
- **`load()`** — writes the CSV to `/data/output.csv` on the Volume and calls `volume.commit()` to make it durable.
- **`run_pipeline()`** — orchestrates the three phases and returns a summary dict.

### `scheduled_etl` function
Decorated with `schedule=modal.Period(hours=1)` — fires the full pipeline every hour automatically.

### `@app.local_entrypoint()` → `main()`
Calls `run_pipeline.remote()`, then writes all four keys into the Modal Dict:
- `records_extracted` → **100**
- `records_loaded` → **50**
- `pipeline_status` → **success**
- `output_path` → **/data/output.csv**
